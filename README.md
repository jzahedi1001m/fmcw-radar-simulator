# FMCW MIMO Radar Simulation & Tracking Pipeline
This repository contains a high-fidelity FMCW (Frequency-Modulated Continuous Wave) MIMO radar simulator written in Python. It models the complete end-to-end signal processing chain from physical RF hardware behaviors to multi-target tracking.

The framework captures real-world physical constraints, including phase noise, thermal noise, multipath ghosting, and quantization behavior, while providing robust state estimation for dynamic targets.

# System Architecture
The pipeline employs a modular architecture, transforming raw electromagnetic returns into stabilized target tracks.

```text
+-----------------------+     +------------------------+     +-------------------+
|   Raw ADC Data Cube   |     |   DSP Processing Chain |     |   GNN Tracking    |
| (Noise + Interference)|     | (FFT, CFAR, ESPRIT)    |     | (Kalman Filters)  |
+-----------+-----------+     +-----------+------------+     +---------+---------+
            |                             |                          |
            v                             v                          v
+-----------------------+     +------------------------+     +-------------------+
|  Environment & Plant  +---->+   Range-Doppler Map    +---->+  Confirmed Track  |
|  (Targets & Clutter)  |     | (Clutter Suppression)  |     |  State Estimates  |
+-----------------------+     +------------------------+     +-------------------+

```

### 1. Radar Frontend & Physical Signal Generation: 
* Models per-chirp phase noise, $kTB$ thermal noise floors, and hardware-level ADC quantization (12-bit).
* MIMO Synthesis: Implements TDM (Time Division Multiplexing) MIMO phase compensation to synthesize a virtual aperture, enabling high-resolution spatial estimation.
* Interference Engine: Generates environmental complexities including direct leakage, bumper reflections, and asynchronous radar interference.

### 2. DSP & Detection Pipeline: 
* Utilizes a Chebyshev/Blackman windowing combination to optimize range sidelobe suppression.
* 2D-CFAR: Implements a Constant False Alarm Rate detector with constant-value padding to eliminate boundary ghost hits.
* ESPRIT DOA: Employs high-resolution ESPRIT (Estimation of Signal Parameters via Rotational Invariance Techniques) with forward-backward spatial smoothing for sub-degree angle estimation.

### 3. State Estimation & Filtering: 
* Utilizes the Joseph form of the Kalman covariance update equation to guarantee positive semi-definiteness, ensuring long-term filter stability.
* Mahalanobis Gating: Applies a 3.5-Sigma Mahalanobis Innovation Validation Gate to filter transient measurement outliers.

### 4. Data Association with Global Nearest Neighbor (GNN): 
Utilizes the Hungarian Algorithm (scipy.optimize.linear_sum_assignment) to perform optimal global assignment, preventing track swapping during target crossings or proximity events.

## Simulation Environment & Scenario Design
The simulation utilizes a custom Environment class designed to stress-test the radar's detection and tracking pipeline against complex, real-world automotive conditions.

### Target Composition
The scene represents a dense urban environment featuring:

* Dynamic Agents: A pedestrian, a cyclist, and a heavy truck, each with distinct radar cross-sections (RCS) and velocity vectors.

* Stationary Infrastructure: Multiple parked cars and clutter objects (trash cans, utility poles) to evaluate the system's ability to resolve closely-spaced targets and distinguish between moving objects and static clutter.

### Advanced Artifact Simulation
The environment goes beyond simple point-target generation by including:

* Multipath Ghosting:  A dynamic ghost target is generated by calculating the signal path reflection of the cyclist off the parked vehicle infrastructure. This allows for rigorous validation of the tracker's data association logic.

* Interference Modeling: The simulator injects both Asynchronous (different slope/carrier frequency) and Synchronous interference, forcing the CFAR (Constant False Alarm Rate) detector to maintain detection performance in degraded signal-to-interference-plus-noise ratio (SINR) conditions.

### Temporal Dynamics
The Environment.update(dt) method provides a continuous state evolution model. By propagating targets forward in time, the simulation generates realistic, frame-to-frame kinematic transitions, enabling the evaluation of the Kalman filter’s state estimation accuracy and ID persistence throughout the duration of the scan.
